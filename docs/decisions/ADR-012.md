# ADR-012: shadcn/ui als Component Library

**Status:** Accepted
**Date:** 2026-02-26
**Deciders:** Roosevelt Digital Team

## Context

Roosevelt OPS gebruikt Tremor v3 voor dashboard UI-components (charts, cards, metrics). Tremor v3 is echter niet meer actief onderhouden: de maintainers hebben het project gestopt en raden zelf aan te migreren naar alternatieven. Dit betekent:

- Geen security fixes voor toekomstige CVEs
- Geen compatibiliteitsupgrades voor nieuwe versies van React/Next.js
- Geen bugfixes
- Groeiende schuld naarmate de rest van de stack evolueert

We moeten een nieuwe component library kiezen die:
- Actief onderhouden wordt
- Tailwind-gebaseerd is (sluit aan bij onze huidige styling aanpak)
- Toegankelijkheid (a11y) serieus neemt
- Goed integreert met Next.js App Router
- Aanpasbaar is aan het Roosevelt design system

## Decision

We migreren van **Tremor v3** naar **shadcn/ui**.

shadcn/ui is geen traditionele component library die je installeert als npm package. In plaats daarvan kopieer je de broncode van components direct in je project. Dit geeft volledige controle zonder upstream afhankelijkheden.

**Aanpak:**
- Components leven in `packages/ui/src/components/`
- Gebaseerd op Radix UI primitives (toegankelijkheid) + Tailwind CSS (styling)
- Aanpassen van components aan het Roosevelt design system is triviaal (broncode in eigen repo)
- Chart-components via `shadcn/ui` + `Recharts` als vervanging voor Tremor charts

**Rationale:**
- Geen package afhankelijkheid: componenten zijn onze eigen code, geen vendor lock-in
- Uitstekende toegankelijkheid via Radix UI primitives
- Tailwind-native: geen CSS-modules of styled-components friction
- Actief onderhouden met grote community
- Next.js App Router officially supported
- Design system aanpassingen zijn triviaal (het is gewoon onze code)

## Consequences

### Positief
- Eliminatie van een deprecated dependency (security risico opgelost)
- Volledige controle over component implementaties
- Uitstekende a11y out-of-the-box via Radix primitives
- Design system customisatie zonder workarounds
- Geen versie-lock in: components evolueren op ons tempo

### Negatief / Trade-offs
- Eenmalige migratie-inspanning (alle Tremor components vervangen)
- Componenten in eigen codebase betekent eigen verantwoordelijkheid voor bugfixes
- shadcn/ui heeft geen eigen chart library: Recharts integratie vereist extra configuratie
- Copy-paste model betekent dat upstream verbeteringen niet automatisch worden overgenomen

## Alternatieven overwogen

- **Tremor v3 behouden:** Geen activeonderhoud, groeiend security risico. Afgewezen.
- **Radix UI direct:** Maximale controle, maar geen styling afspraken. Vereist volledig eigen design system implementatie. Te arbeidsintensief. Afgewezen.
- **Material UI (MUI):** Uitgebreid en goed onderhouden, maar sterke Material Design esthetiek die conflicteert met het Roosevelt brand. Zware bundle size. Afgewezen.
- **Mantine:** Uitstekende library maar Tailwind-integratie is suboptimaal (eigen CSS variabelen systeem). Afgewezen vanwege styling-conflicten met onze Tailwind-first aanpak.

# ROOSE-108: Contextual Inquiry — 5 Sessies bij Target Bedrijven

**Plane Issue:** ROOSE-108 (sub van ROOSE-106)
**Type:** Research Schema
**Prioriteit:** High
**Budget:** €500-1.000 (reiskosten + incentives)
**Doorlooptijd:** 3-4 weken

---

## Doel

Observeer en interview 5 ops managers/founders bij NL SMBs in hun werkomgeving om te begrijpen hoe ze momenteel operationele metrics verzamelen, analyseren en gebruiken voor besluitvorming.

---

## Methodologie: Contextual Inquiry (Beyer & Holtzblatt)

### Vier Principes

| Principe | Toepassing Roosevelt OPS |
|----------|--------------------------|
| **Context** | Op locatie bij de klant, niet in een meeting room |
| **Partnership** | Master-apprentice model: "Leer mij hoe jij dit doet" |
| **Interpretation** | Deel interpretaties realtime: "Dus je zegt dat..." |
| **Focus** | Specifiek op operationele metrics en besluitvorming |

### Sessie Structuur (3 uur per sessie)

| Fase | Duur | Activiteit |
|------|------|-----------|
| **1. Primer** | 15 min | Introductie, consent, context project uitleg |
| **2. Office Tour** | 15 min | Rondleiding, observeer tools op schermen, Post-its, dashboards |
| **3. Shadowing** | 60 min | Volg de deelnemer tijdens operationeel werk |
| **4. Deep Interview** | 45 min | Doorvragen op geobserveerde patronen |
| **5. Tool Demo** | 30 min | Laat deelnemer tools laten zien die ze gebruiken |
| **6. Wrap-up** | 15 min | Samenvatting, vragen, bedanken |

## Observatie Protocol

### Wat te Observeren

| Categorie | Specifiek Zoeken Naar |
|-----------|----------------------|
| **Huidige tools** | Spreadsheets, Jira, Asana, Notion, custom dashboards, Datadog, Grafana |
| **Data verzameling** | Handmatig vs. geautomatiseerd, frequentie, bronnen |
| **Rapportage flow** | Wie maakt rapporten? Voor wie? Hoe vaak? |
| **Decision triggers** | Welke metrics leiden tot actie? Welke worden genegeerd? |
| **Pijnpunten** | Copy-paste werk, tool switching, data inconsistenties |
| **Workarounds** | Creatieve oplossingen voor tool beperkingen |
| **Integratie gaps** | Welke tools praten niet met elkaar? |
| **Tijd besteding** | Hoeveel uur/week aan operationele rapportage? |

### Interview Guide (Master-Apprentice Model)

**Opening:**
- "Kun je me laten zien hoe een typische maandag begint qua operationeel werk?"
- "Welke dashboards of rapporten bekijk je als eerste?"

**Dagelijkse workflow:**
- "Loop me door je proces van data verzamelen tot rapportage"
- "Wie vraagt om deze informatie en hoe vaak?"
- "Wat gebeurt er als een metric onder/boven een drempel komt?"

**Tools & integraties:**
- "Laat me zien welke tools je open hebt staan"
- "Hoe komen gegevens van [Tool A] naar [Tool B]?"
- "Waar kopieer je handmatig data tussen systemen?"

**Pijnpunten:**
- "Wat kost je het meeste tijd in dit proces?"
- "Waar maak je de meeste fouten?"
- "Als je een toverstaf had, wat zou je veranderen?"

**Besluitvorming:**
- "Geef een voorbeeld van een recente beslissing gebaseerd op deze data"
- "Hoe snel na een incident heb je de data beschikbaar?"
- "Welke informatie mis je om betere beslissingen te nemen?"

## Documentatie & Analyse

### Tijdens de Sessie

- **Audio opname** (met consent) + foto's van werkplekken/schermen
- **Notities** in gestructureerd template (zie hieronder)
- **Artefacten verzamelen**: Screenshots van tools, exports van rapportages

### Na de Sessie: Debrief Template

```markdown
## Sessie [#] — [Bedrijfsnaam]

**Datum:** [datum]
**Deelnemer:** [naam], [rol], [bedrijfsgrootte]
**Sector:** [sector]

### Key Observations
1. [Observatie + interpretatie]
2. [...]

### Tools in Gebruik
| Tool | Doel | Frequentie | Tevredenheid (1-5) |
|------|------|-----------|-------------------|

### Pijnpunten (gerankt)
1. [Pijnpunt] — ernst: hoog/medium/laag
2. [...]

### Workarounds
1. [Wat doen ze nu als oplossing?]

### Quotes (verbatim)
- "[Quote]" — context: [...]

### Implications voor Roosevelt OPS
- [Wat betekent dit voor ons product?]
```

### Synthese (na alle 5 sessies)

1. **Affinity Diagram** — Cluster alle observaties in thema's
2. **Flow Model** — Visualiseer de operationele data flow per bedrijf
3. **Consolidated Findings** — Top 5 pijnpunten met frequentie
4. **Persona Refinement** — Update buyer persona op basis van observaties
5. **Feature Priority** — Welke features addressen de meest geobserveerde pijn?

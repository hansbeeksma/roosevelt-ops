# ADR-008: Build vs Buy Strategy

**Status:** Accepted
**Date:** 2026-02-26
**Deciders:** Roosevelt Digital Team

## Context

Roosevelt Digital heeft een intern operations platform nodig dat CRM, project management, time tracking, client portal, en AI-ondersteunde operaties ondersteunt. De kernvraag: welke onderdelen kopen we als SaaS/open-source, en welke bouwen we zelf?

De afweging is altijd: **TCO (Total Cost of Ownership) vs strategische differentiatie**. Generieke functies (CRM-basisflows, boekhouding) zijn geen concurrentievoordeel — we bouwen ze niet zelf. Functies die ons werkproces uniek maken (design cycle management, AI-assisted ops) bouwen we wel.

**TCO analyse (jaar 1):**
| Strategie | Jaar 1 | Jaar 2+ |
|-----------|--------|---------|
| Hybrid (gekozen) | €25.000–€35.000 | €8.000–€16.000 |
| Full SaaS | €8.000–€11.000 | €8.000–€11.000 |
| Full Custom | €52.000–€89.000 | €15.000–€25.000 |

## Decision

We kiezen een **hybride aanpak**: koop commodity functionaliteit, bouw differentiërende features.

### KOPEN (Buy)

| Tool | Functie | Motivatie |
|------|---------|-----------|
| **Twenty CRM** | Client & deal management | Open-source, self-hosted, MCP-integratie |
| **Plane** | Project & issue management | Al in gebruik, rijke API, 76 MCP tools |
| **Exact Online** | Boekhouding & facturatie | NL-compliance, accountant integratie |
| **Inngest** | Event-driven workflows | Managed durable execution (zie ADR-007) |

### BOUWEN (Build)

| Module | Motivatie |
|--------|-----------|
| **Time Tracking** | Geïntegreerd met Plane issues, custom approval flows |
| **Client Portal** | Branded ervaring, custom design review flows |
| **Design Cycle Manager** | Uniek aan Roosevelt workflow, geen SaaS equivalent |
| **AI Ops** | Proprietary agent orchestratie, differentiërend |
| **Integratie Layer** | Verbindt Twenty ↔ Plane ↔ Exact ↔ Slack |

**Rationale voor hybrid:**
- Directe operationele waarde: Twenty en Plane zijn vandaag bruikbaar
- Geen heruitvinden van het wiel voor commodity functies
- Bouwbudget geconcentreerd op differentiërende features
- Open-source tools (Twenty) geven controle over data zonder vendor lock-in

## Consequences

### Positief
- Lagere initiële investering dan full-custom
- Snellere time-to-value voor het team
- Data soevereiniteit via self-hosted opties (Twenty, Plane)
- Flexibiliteit om SaaS later te vervangen als behoeften veranderen

### Negatief / Trade-offs
- Integratie-complexiteit: meerdere systemen moeten samenwerken
- Hogere jaar-1 kosten dan pure SaaS-strategie
- Risico op "integration hell" bij incompatibele API-updates van externe tools
- Onderhoudslast voor self-hosted componenten (Twenty op Hetzner)

## Alternatieven overwogen

- **Full SaaS (HubSpot + Jira + Harvest + etc.):** Laagste kosten, maar geen controle over data, geen custom workflows, geen MCP-integratie. Beperkt schaalbaarheid van AI-features. Afgewezen vanwege strategische lock-in.
- **Full Custom:** Maximum controle en differentiatie, maar €52k–€89k jaar 1 is niet gerechtvaardigd voor interne tooling. Time-to-value te lang. Afgewezen.
